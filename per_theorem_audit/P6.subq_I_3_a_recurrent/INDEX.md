# P6.subq_I_3_a_recurrent — per-theorem deep dive

**Total PASSes**: 115 (53 Goedel + 62 Kimina)

## Statement

Show that `a_n = 2^n + 3^n` is generated by a linear recurrence: exhibit `r : ℕ` and
`q : Fin (r+1) → ℝ` with `q 0 ≠ 0` such that `∑_k q k • (2^(n+r-k) + 3^(n+r-k)) = 0` for all `n`.

## Corrigé route (Mercier-Rombaldi)

Characteristic polynomial of `{2,3}` is `(X−2)(X−3) = X²−5X+6`. Witnesses: `r = 2`,
`q = (6, −5, 1)` (or equivalently `(1, −5, 6)` depending on indexing convention).
Recurrence verification closes by ring expansion: `(4−10+6)·2^n + (9−15+6)·3^n = 0`.
Named moves: (1) compute char. poly, (2) identify witnesses, (3) ring-expand per-`n` equation.

## Sub-styles per arm

### Goedel — all (b) lower-abstraction

Every Goedel PASS wraps the body in a `have h_main := by ... ; exact h_main` scaffold,
uses `use 2` + `Fin.sum_univ_succ` to unfold the three-term sum, then closes with a
multi-step cascade: `ring_nf <;> norm_num <;> simp_all [...] <;> linarith [pow_pos ...]`.
The cascade fires multiple tactic fallbacks sequentially; no single clean tactic closes the goal.
Three witness-notation sub-styles occur, all with the same shotgun close:

- **G-b1** (29 files): `![1, −5, 6]` vector notation (or scalar multiple `![6, −30, 36]` in att50).
  See `goedel_att05.md` (concise, 33 lines), `goedel_att11.md`.
- **G-b2** (10 files): `fun i => match i with | ⟨0, _⟩ => 1 | ⟨1, _⟩ => −5 | ⟨2, _⟩ => 6 | ⟨n+3, _⟩ => by exfalso; omega`.
  See `goedel_att00.md`, `goedel_att07.md`.
- **G-b3** (14 files): `fun i => if i.val = 0 then 1 else if i.val = 1 then −5 else 6`.
  See `goedel_att04.md`, `goedel_att58.md`.

In all three sub-styles the sum-unfolding hook is `Fin.sum_univ_succ` (not `Fin.sum_univ_three`,
not `Finset.sum_range_succ`), and the arithmetic close never reduces to a single `ring`.

### Kimina — (a) route-preserving (61/62)

Kimina's canonical script is 17–18 lines with no wrapper and no shotgun:
`use 2`, `use fun k => match k with | 0 => 1 | 1 => −5 | 2 => 6 | _ => 0`,
`simp [Finset.sum_fin_eq_sum_range, Finset.sum_range_succ]`, `ring_nf`.
The `ring_nf` call closes the goal after `simp` reduces the `Finset` sum to three explicit terms.
The char-len distribution is tightly clustered (463–604 chars) confirming near-identical scripts.

Two minor variants within (a):

- **K-a1** (59 files): `simp [...sum_range_succ] <;> ring_nf`. Canonical form. See `kimina_att00.md`.
- **K-a2** (2 files): same structure with an extra `norm_num` or `all_goals ring` appended
  (att36, att44, att51). See `kimina_att51.md` (21 lines).

### Kimina — (c) non-canonical (1/62)

- **K-c1**: `kimina_att31.md` — `r = 3`, witnesses `(1, 0, −19, 30)`. The CoT attempts to solve
  a 4×4 linear system numerically and lands on a non-minimal recurrence. Script closes identically
  (`simp [sum_range_succ] <;> ring_nf`) but the choice of `r` diverges from the corrigé.

## Cross-arm divergence

The two arms occupy disjoint sub-strategy spaces. Kimina (98%) clusters on a single compact
proof pattern (`sum_range_succ` + `ring_nf`) that directly mirrors the corrigé's algebraic move.
Goedel (100%) uses `Fin.sum_univ_succ` to unfold the three-term sum and then discharges with
a cascaded arithmetic shotgun (`ring_nf+norm_num+simp_all+linarith`). The Goedel scripts are
1.5–6× longer (884–2773 chars vs 463–604 chars for Kimina). No Goedel PASS uses
`Finset.sum_range_succ`; no Kimina PASS uses `Fin.sum_univ_succ` or `simp_all`.

## Bidirectionality verdict

For Kimina, the named corrigé moves — identify witnesses (1,−5,6), unfold the sum, close by ring
expansion — are directly recoverable from the tactic script; the script is a near-literal
transcription of the three-step corrigé argument. For Goedel, the witnesses are present and
semantically correct, but the closing tactic is an opaque cascade whose individual steps do not
correspond to named mathematical operations; a reader cannot reconstruct "4−10+6=0 and 9−15+6=0"
from the script without running it. The strong arm split (Kimina 98% (a) vs Goedel 100% (b))
on a theorem with a uniquely determined minimal witness provides a clean instance of
within-difficulty epistemic divergence: both arms find the same mathematical object but differ
sharply in whether the proof term encodes the corrigé reasoning.
