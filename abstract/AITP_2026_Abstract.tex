\section{The metric}

Pass rate measures one thing: whether the kernel accepts the script. Bidirectionality measures another: whether a Lean-literate mathematician can reconstruct the proof's case structure, named lemmas, and strategy from the script alone. The notion is developed at length in~\cite{appadourai-wdwft}; here, with a paired official solution, it becomes per-PASS computable. Each named solution move appears in the script as a named operation (\texttt{have h\,:\,T := proof}, \texttt{apply <named\_lemma>}) or is absorbed into closing automation (\texttt{nlinarith [shotgun]}, \texttt{<;> field\_simp <;> ring\_nf}). Three discrimination levels emerge: \emph{(i)} move-level inscription (each solution move appears as \texttt{have}); \emph{(ii)} tactic-level inscription (closure by named lemma vs.\ \texttt{fun\_prop} / \texttt{<;>} cascade); \emph{(iii)} inscription-uniqueness (one \texttt{have} per move). These collapse into three per-PASS bins: (a) route-preserving, (b) route-equivalent at lower abstraction, (c) neither: opaque automation, or a script that closes a different problem than the corrig\'e.

\section{Setup}

Corpus: agr\'egation interne 2005--2013~\cite{mercier-rombaldi}, a French print-only academic publication absent from training corpora (Numina, Lean-Workbook~\cite{leanworkbook}, AOPS) and benchmarks (miniF2F, PutnamBench, ProofNet) by documented-source enumeration; none index French print-only academic publications. Web pre-training contamination of the LLM bases is very low-probability: Publibook print-on-demand, out of library circulation. From 412 audited sub-questions, 20 focal theorems autoformalised in Lean 4 (Mathlib v4.30.0-rc2): 7 algebra, 13 analysis. Selection: pre-experiment 4-band stratification at $K{=}8$ pilot, crossed with algebra/analysis balance, proof scripts not inspected pre-selection. Provers: Goedel-Prover-V2-32B (RL closure-tuning,~\cite{goedelv2}) and Kimina-Prover-72B (RL with \texttt{<think>}-block informal-formal alignment,~\cite{kimina}). Both arms ran at $K = 64$, max\_tokens $= 32{,}000$, top-$p = 0.95$ (Goedel $T=1.0$, Kimina $T=0.6$ per published protocols; vLLM seed unset on the main run, secondary seeds 12345 and 67890 in \S5), with one operational exception: 50 of Kimina's 64 attempts on P11.subq\_III\_1 were not run after the first 14 each took 27--55 minutes (corpus median $<2$ min) and would have exhausted compute budget for the remaining 19 theorems; Goedel ran the full $K=64$ on this theorem. Total: 2,510 completed attempts (1,280 Goedel + 1,230 Kimina), 805 PASSes (520 + 285). Per-PASS classification was hand-mediated through Sonnet sub-agents calibrated on representative samples.

\section{Findings}

The bidirectional gap is signed, sizable, and concentrated. \emph{First,} per-PASS classification of all 805 verified scripts yields Kimina 147 (a) / 137 (b) / 1 (c) and Goedel 102 (a) / 418 (b) / 0 (c). Kimina preserves the solution route at 51.6\% per PASS, Goedel at 19.6\%, despite Goedel's 1.83$\times$ higher PASS rate (520 vs.\ 285). The gap concentrates in seven theorems where Kimina's outputs name solution moves explicitly and Goedel's outputs absorb them into closing automation.

\emph{Second,} the showcase theorem P12 ($x^2+xy+y^2 \geq (3/4)x^2$ via $(x/2+y)^2 \geq 0$) inverts the corpus-wide gap direction: Goedel 55/56 (a), Kimina 9/63 (a). The (a)\% ordering is per-theorem, not constant across the corpus.

\section{The two ceilings}

Two theorems produce 0 PASSes at paper-protocol budget: P11.subq\_III\_1 ($g$ differentiable on $]p,q[$ with $|g'| \leq M \Rightarrow g$ has a finite limit at $q^-$; two-line official solution via MVT + Cauchy criterion; 0/78 completed attempts, see \S2) and P9.subq\_22 (winding number invariance under reparameterisation; 0/128 attempts after Lake-FP exclusion). The single Kimina PASS originally listed for P9.subq\_22 is a Lake false-positive: the script body terminates at \texttt{:= by} with no closing tactic. \texttt{lake env lean} in batch mode accepts the malformed proof (its exit code tracks parse-success, not goal closure), while an interactive lean-lsp re-check on the same Mathlib v4.30.0-rc2 environment rejects it. The two false-positives reported below are pipeline-level artifacts of this kind, not Lean kernel false-positives.

To localise the failures, a Claude Opus 4.7 + lean-lsp-mcp transcription of the official solution was produced for all 20 theorems via interactive lean-lsp (proof-state inspection, multi-tactic probing, Mathlib API search). Of 20 baseline scripts, 19 reach (a) within four iterations (mean 1.3, median 1). P11.subq\_III\_1 closes in one iteration via MVT + Cauchy criterion (named Mathlib lemmas; see repo). The Mathlib API is complete. The failure is consistent with premise-selection. P9.subq\_22 also bounces the baseline, but with a structured diagnosis: the Lean statement does not fix $\varphi(0) = 0$ (the official solution silently does), and the period-shift invariance of the rotation index (Q21 of the original problem) is absent from Mathlib v4.30.0-rc2. \emph{In short,} the first ceiling is provers' weakness, the second a content gap.

\section{Limitations}

The score is corpus-relative. Whether the pattern survives in other traditions is open. The $K{=}64$ main run is single-seed at the prover level. On a Goedel 3-theorem stratified subset (concrete, standard, abstract), two secondary vLLM seeds (12345, 67890) reproduce a regex proxy preceding the \S3 hand-classification within $\leq 3.2$\,pp; kernel-PASS rates vary by 9--14\,pp. Hand classification was not multi-seeded; whether it would be similarly stable is open. Full multi-seed coverage and Kimina re-seeding deferred. Per-PASS classification is sub-agent-mediated, in the same sense that referee judgment of readability is. A 3-rater protocol (Sonnet original, Gemini 3.1 Pro, GPT-5.5; three independent model families) achieves Fleiss' $\kappa = 0.87$ across 52 valid triples; excluding P16.subq\_IV\_3 (slope/multiplicative bound), all three agree on every PASS (n=37). The (a)/(b) boundary is fuzzy on that one criterion; unanimous across model lineages elsewhere. All 805 PASSes were verified via \texttt{lake env lean} on Mathlib v4.30.0-rc2; two Lake false-positives surfaced under interactive lean-lsp re-verification (both Kimina; details in repo), remain in the 805 denominator, and shift Kimina's per-PASS route rate by less than 0.5\,pp on exclusion. The transcription baseline is itself Claude+lean-lsp-mediated; expert-authored comparison is future work.
